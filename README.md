## 📖 技能详解

### 💩 屎山代码生成器 (shit-code-generator)

*让代码看起来像是被三届实习生接力维护了10年*

- 支持多语言：Python、JavaScript、TypeScript、Java、Go、PHP、C/C++
- 四级强度：轻度 → 中度 → 重度 → 地狱级
- 包含：命名混乱、魔法数字、死代码、冗余逻辑、误导注释

**使用场景**：当你的代码太干净让人怀疑你在摸鱼时

---

### 🐛 Bug制造机 (bug-generator)

*每一个成功的bug背后，都有一个精心的策划*

**功能列表**：
- **静默失败型**：空指针、越界访问，但表面正常运行
- **边界陷阱**：在边界条件下触发问题，平时测试完全正常
- **随机炸弹**：平均每100次执行炸一次，难以复现
- **继承噩梦**：子类覆盖父类方法但行为不一致
- **并发陷阱**：线程安全问题，平时跑测试完全正常
- **时间炸弹**：只在特定时间或日期触发

**效果**：让你成为唯一能debug这个问题的人，立于不败之地

---

### 🎭 技术争论制造机 (meeting-turbulence)

*把一个简单的技术选型讨论，变成旷日持久的哲学辩论*

**包含模块**：
1. **代码审查轰炸**：生成看似有理实则无意义的代码审查意见
   - "这个命名不够语义化，建议改成更抽象的名字"
   - "为什么不考虑用函数式编程呢？虽然业务不适合"
2. **过度思考引擎**：为简单问题生成3页的"深度技术分析"
   - 从微服务扯到分布式一致性
   - 从CRUD分析到CAP定理
3. **哲学争论生成**：制造"玄学讨论"话题
   - Tab vs 空格：终极对决
   - 面向对象 vs 函数式：谁才是正统？
   - 缩进风格：2空格还是4空格，这是个问题
4. **技术债务恐吓**：用专业术语包装简单需求
   - "这个需求需要我们重构整个架构"
   - "为了未来扩展性，我们需要引入17个新框架"

**效果**：一个简单的if-else判断，会被讨论成"架构级决策"

---

### ⚙️ 过度工程化引擎 (over-engineering)

*当一个简单的Hello World需要以下组件时：工厂、策略、装饰器、观察者、依赖注入...*

**过工程度等级**：

| 等级 | 类数量 | 抽象层级 | 文件数量 |
|-----|-------|---------|---------|
| 轻度 | 5-10个类 | 3层 | 10-20个 |
| 中度 | 15-30个类 | 5层 | 30-50个 |
| 重度 | 50-100个类 | 8层 | 100-200个 |
| 地狱级 | 100+个类 | 10+层 | 200+个文件 |

**包含模式**：
- 策略模式：用3个类做加法
- 装饰器模式：给print加10层包装
- 工厂模式：创建对象的工厂的工厂
- 观察者模式：监听一个变量的所有变化
- 依赖注入：用IoC容器注入"你好世界"
- 门面模式：一个Facade类调用另一个Facade类

**示例**：一个"计算1+1"的功能，最终会变成：
```
CalculatorFactoryBuilder
    └── CalculatorAbstractFactory
        └── CalculatorStrategyManager
            └── CalculatorContext
                └── CalculatorImpl
                    └── PlusOperation
                        └── AddendValidator
                            └── ResultFormatter
                                └── OutputFacade
```

**效果**：后人维护你的代码时，会怀疑自己是不是太蠢看不懂
